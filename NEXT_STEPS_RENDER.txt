Arquivos que parecem realmente interessantes para o projeto
-----------------------------------------------------------
1) snppublend_c
   - Blender/render path em C/C++
   - Bom para entender a lógica de blend da PPU
   - Deve servir como referência/base segura

2) snppublend_mm
   - Blender otimizado para PS2/EE
   - Provavelmente usa instruções vetoriais/MMI
   - Útil como referência de otimização
   - Precisa checar compatibilidade com headers/layout atuais

3) snppublend_gs
   - Muito importante investigar
   - No fork analisado, parece ser o backend PS2 realmente usado
   - Pode ser mais importante que snppublend_mm no estado atual

Arquivo que NÃO é de render
---------------------------
- snspc700
  - Está ligado ao SPC700 (áudio)
  - Não é prioridade para a fase atual de render
  - Pode ser revisto depois, em etapa de áudio/performance

Ideia de estratégia
-------------------
Não tentar puxar tudo de uma vez.
O melhor caminho parece ser:
- estudar primeiro
- entender dependências
- integrar por etapas pequenas
- compilar e testar a cada etapa
- salvar checkpoints no GitHub antes de mudanças maiores
